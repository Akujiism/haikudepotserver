Versionsübersetzungen sind Übersetzungen eines Pakets, deren Inhalt sich zwischen Versionen und Architekturen dieses Pakets unterscheiden können. Dadurch unterscheiden sie sich von normalen Übersetzungen, die für alle Versionen und Architekturen gleich sind.

Die folgende Seite zeigt die Übersetzungen für eine Version + Architektur eines Pakets. Auf der linken Seite befinden sich die verschiedenen Sprachen, an denen gearbeitet wird. Ein Klick auf eine Sprache zeigt die Übersetzungen für diese Sprache. Die verschiedenen Übersetzungen können bearbeitet und mit dem Aktions-Button ganz unten gespeichert werden.

Die gleiche Übersetzung lässt sich auf alle Architekturen derselben Paket + Versions Kombination anwenden. Gibt es beispielsweise das Paket "deen" und hat dieses Paket mehrere Versionen...

  • deen - 1.2.2 - x86_gcc2
  • deen - 1.2.2 - x86_64
  • deen - 1.2.3 - x86_gcc2
  • deen - 1.2.3 - x86_64

...und man bearbeitet die Übersetzung für "deen - 1.2.3 - x86_gcc2" und wählt die Option der Übertragung auf alle Architekturen, dann würde die Übersetzung auf folgende Pakete angewandt werden:

  • deen - 1.2.3 - x86_gcc2
  • deen - 1.2.3 - x86_64

Die englischsprachigen Texte können nicht editiert werden, da sie direkt aus den Paketdaten stammen.

Werden neue Versionen eines Pakets in das Depot importiert, werden die Übersetzungen automatisch übernommen, sofern sich die englischsprachigen Texte zur vorigen Version nicht verändert haben.
